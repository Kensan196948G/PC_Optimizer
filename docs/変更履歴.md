## [4.0.1] - 2026-03-05

### 修正

- `modules/Advanced.psm1`: PS5.1 で `Invoke-RestMethod` が Anthropic API の UTF-8 レスポンスを
  システムデフォルトエンコーディング（ANSI/CP932）で誤デコードして AI 診断サマリー等が文字化けになる
  バグを修正。PS5.1 では `Invoke-WebRequest` + `RawContentStream` + 明示的 UTF-8 デコードに切り替え、
  PS7 以上は従来の `Invoke-RestMethod` のまま（正常動作）。
- `modules/Orchestration.psm1`: Hook の `command` 型で `Invoke-Expression` を使用して
  設定ファイル由来の任意コードを実行できる脆弱性を修正。`command` 型 hook は廃止し、
  `webhook`/`file` 型のみをサポート。
- `PC_Optimizer.ps1`（`Clear-DirContents`）: `cmd.exe` + ダブルクォート付き引数によるコマンド
  インジェクションを修正。`Remove-Item -LiteralPath` による安全なネイティブ削除に変更。
- `PC_Optimizer.ps1`（`Get-SystemDriveFreeGB`）: `$disk.FreeSpace -ne $null` の比較順序が
  配列比較と誤解釈されるリスクを修正。`$null -ne $disk.FreeSpace` に変更。
- `PC_Optimizer.ps1`（電源プラン最適化）: `powercfg /setactive` の `$LASTEXITCODE` が
  ノートPC分岐で変数代入後まで保留されていた問題を修正。コマンド直後に `$powercfgExit` に
  キャプチャするよう変更。
- `PC_Optimizer.ps1`（ディスク最適化）: `defrag` 終了コードの未チェックを修正。
  非ゼロ終了コード時にログ出力を追加。
- `modules/Common.psm1`, `Advanced.psm1`, `Orchestration.psm1`, `Report.psm1`:
  ファイル書き込みで `-Encoding utf8`（ハードコード）を使用していた規約違反を修正。
  各モジュール冒頭に `$script:_enc` を定義（PS7: `utf8NoBOM`, PS5.1: `UTF8`）し、
  全書き込み操作で使用するよう統一。

### 追加
