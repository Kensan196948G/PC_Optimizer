# 使い方

## 基本的な実行手順

### BAT ファイル経由（推奨）

1. `Run_PC_Optimizer.bat` を右クリック
2. **「管理者として実行」** を選択
3. UAC（ユーザーアカウント制御）画面で **[はい]** をクリック
4. 処理が自動で開始される
5. 全処理完了後、`Enter` キーで終了

### PowerShell から直接実行

```powershell
# 管理者権限の PowerShell コンソールで実行
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\path\to\PC_Optimizer.ps1"

# PowerShell 7 の場合
pwsh -NoProfile -ExecutionPolicy Bypass -File "C:\path\to\PC_Optimizer.ps1"
```

## 実行タスクの順序（20 タスク）

| # | タスク | 説明 |
|---|---|---|
| 1 | 一時ファイルの削除 | `C:\Windows\Temp`, `%TEMP%`, `%USERPROFILE%\AppData\Local\Temp` |
| 2 | Prefetch・更新キャッシュの削除 | Prefetch, SoftwareDistribution |
| 3 | 配信最適化キャッシュの削除 | `SoftwareDistribution\DeliveryOptimization\Cache` |
| 4 | Windows Update キャッシュの削除 | `SoftwareDistribution\Download` |
| 5 | エラーレポート・ログ削除 | WER ReportArchive/Queue, CBS ログ |
| 6 | アプリキャッシュ削除 | OneDrive ログ, Teams キャッシュ, Office キャッシュ |
| 7 | ブラウザキャッシュの削除 | Chrome / Edge / Firefox 全プロファイル対応 |
| 8 | サムネイルキャッシュの削除 | `thumbcache_*.db` |
| 9 | Microsoft Store キャッシュのクリア | LocalCache / LocalState\Cache |
| 10 | ごみ箱を空にする | `rundll32.exe` 経由、全ドライブ対象 |
| 11 | DNS キャッシュのクリア | `ipconfig /flushdns` |
| 12 | Windows イベントログのクリア | Application / System（Security は対象外） |
| 13 | ディスクの最適化 | SSD → TRIM、HDD → デフラグ（自動判定） |
| 14 | SSD 健康診断 | SMART ステータス・モデル・シリアル番号 |
| 15 | システムファイルの整合性チェック・修復 | `sfc /scannow` |
| 16 | Windows コンポーネントストアの診断 | `DISM /Online /Cleanup-Image /ScanHealth` |
| 17 | 電源プランの最適化 | ノートPC: バランス / デスクトップ: 高パフォーマンス |
| 18 | **Microsoft 365 の更新確認・適用** ★ | バージョン表示 → **Y/N 確認** → `OfficeC2RClient.exe` で更新 |
| 19 | **Windows Update の確認・適用** ★ | 更新一覧表示 → **Y/N 確認** → `UsoClient.exe` で適用 |
| 20 | スタートアップ・サービスレポート | スタートアップ登録アプリ一覧表示・不要サービス稼働チェック（読み取り専用） |

★ マークのタスクはユーザーが Y/N で実行可否を選択できます。

## 更新確認プロンプト

タスク 18・19 では更新内容を表示した後、Y/N で適用の可否を選択できます。

### Microsoft 365 更新確認（タスク 18）

```
Microsoft 365 を検出しました。
  現在のバージョン: 16.0.18827.20160
Microsoft 365 の更新を確認・適用しますか？ (Y/N): _
```

- `Y` → `OfficeC2RClient.exe` で更新を開始（更新 UI が表示される場合があります）
- `N` → スキップして次のタスクへ
- Microsoft 365 がインストールされていない場合は自動スキップ

### Windows Update 確認（タスク 19）

```
Windows Update を確認中... (しばらくお待ちください)
利用可能な更新: 3 件
  [1] 2026-02 x64 ベース システム用 Windows 11 累積更新プログラム (KB5051987)
  [2] Microsoft Defender Antivirus のセキュリティ インテリジェンス更新プログラム
  [3] Windows Malicious Software Removal Tool x64 - v5.130
これらの更新を適用しますか？ (Y/N): _
```

- `Y` → `UsoClient.exe` でダウンロード・インストールを開始
- `N` → スキップして次のタスクへ
- 更新が 0 件の場合はプロンプトを表示せずスキップ

## 再起動プロンプト

Windows Update などで再起動が必要になった場合、全タスク完了後に確認プロンプトが表示されます。

```
再起動が必要です。今すぐ再起動しますか？ (Y/N): _
```

- `Y` → 即座に再起動
- `N` または `Enter` → スキップして終了

## 定期実行の設定（タスクスケジューラ）

月次メンテナンスとして定期実行する場合は、タスクスケジューラで設定します。

1. `taskschd.msc` を開く
2. **タスクの作成** → 全般タブで「最上位の特権で実行する」にチェック
3. トリガー → 毎月・任意の日時を設定
4. 操作 → プログラム: `powershell.exe`
   引数: `-NoProfile -ExecutionPolicy Bypass -File "C:\path\to\PC_Optimizer.ps1"`

## v4.0 CLI 使用例

### 1. タスク範囲指定で実行

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\PC_Optimizer.ps1 -Tasks "1-5,7,10-12"
```

- `-Tasks` は `1,3,7` と `1-5,7,10-12` の両方をサポートします。
- 対象外タスクは `SKIP` として記録されます。

### 2. 非対話 + WhatIf（副作用なし）

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\PC_Optimizer.ps1 `
  -NonInteractive -WhatIf -NoRebootPrompt -Tasks "1-3,7"
```

- 変更系タスクは実行せず、プレビュー結果のみ記録します。

### 3. 削除候補一覧を任意フォルダへ出力

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\PC_Optimizer.ps1 `
  -NonInteractive -WhatIf -Tasks "1-3,7" `
  -ExportDeletedPaths json -ExportDeletedPathsPath "D:\temp\pcopt"
```

- `-ExportDeletedPaths` は `json` / `csv` を指定可能です。
- `-ExportDeletedPathsPath` 未指定時は `logs\` へ保存されます。

### 4. 失敗モードの指定

```powershell
# 失敗しても継続（既定）
powershell -File .\PC_Optimizer.ps1 -FailureMode continue

# 失敗時に後続タスクを実行せず終了
powershell -File .\PC_Optimizer.ps1 -FailureMode fail-fast
```

動作差分:

| モード | タスク失敗時 | 終了コード | レポート |
|---|---|---|---|
| `continue` | 後続タスクを継続実行 | `1`（一部失敗時） | 失敗タスクを `NG` で記録 |
| `fail-fast` | 直ちに中断 | `2` | 中断以降タスクを `SKIP` として記録し、`unexecutedTasks` に出力 |

実行例（差分確認）:

```powershell
# continue: 途中失敗があっても最後まで回す
powershell -File .\PC_Optimizer.ps1 -Tasks "11,13,15" -FailureMode continue

# fail-fast: 最初の失敗で停止
powershell -File .\PC_Optimizer.ps1 -Tasks "11,13,15" -FailureMode fail-fast
```

終了コード:

- `0`: 全処理成功
- `1`: 一部失敗（continue で完走）
- `2`: fail-fast で中断
- `3`: 引数エラー
- `4`: 管理者権限不足

## -Tasks 入力エラーの扱い

`-Tasks` には重複・空要素・末尾カンマを許容しません。いずれも終了コード `3` で終了します。

```powershell
# 末尾カンマ（NG）
powershell -File .\PC_Optimizer.ps1 -Tasks "1,3,7,"

# 空要素（NG）
powershell -File .\PC_Optimizer.ps1 -Tasks "1,,7"

# 重複（NG）
powershell -File .\PC_Optimizer.ps1 -Tasks "1,3,3"
```
