# ClaudeCode 実装憲法 v2.0

## 目的

ClaudeCode による設計 → 実装 → レビュー → 修復を
再現性あるプロセスとして統一する。

---

## 基本原則

| # | 原則 | 内容 |
|---|---|---|
| 1 | 設計なし実装禁止 | 影響範囲分析なしにコードを書かない |
| 2 | Encoding 明示義務 | `Out-File` / `Add-Content` 等は必ず `-Encoding` を指定する |
| 3 | ログは UTF-8 統一 | バージョン分岐で `$logEncoding` 変数を使う |
| 4 | CI 失敗は即修復移行 | 同一エラー 3 回で停止しユーザーに報告 |
| 5 | 無限ループ防止 | 修復ループは最大 15 回 |
| 6 | 既存コード優先 | 新規ファイル作成より既存ファイルの編集を優先する |
| 7 | 最小変更原則 | 要求されていない変更・リファクタリングを行わない |

---

## 開発フェーズ

### Phase 1 — Plan（設計）

- 要件確認・曖昧点は AskUserQuestion で解消
- 既存コードを Read / Grep / Glob で事前探索
- 影響範囲を列挙してから実装方針を決定
- エンコーディング方針を明示

### Phase 2 — Implement（実装）

- エンコーディングを変数化し全ファイル出力に適用
- エラーハンドリングは try-catch で明示（`SilentlyContinue` の乱用禁止）
- コメントは「なぜ」を記述する（「何を」はコードから自明）
- 1 つの関数は 1 つの責任のみ持つ

### Phase 3 — Review（レビュー）

以下の観点を必ず確認する：

| 観点 | 確認内容 |
|---|---|
| セキュリティ | コマンドインジェクション・パス検証・権限昇格 |
| 文字コード | 全 `Out-File` / `Add-Content` に `-Encoding` が付いているか |
| PS バージョン | PS5.1 と PS7 の両方で動作するか |
| Windows 依存 | 使用コマンド・パスが Windows 環境前提になっていないか確認 |
| エラー処理 | 例外が握り潰されていないか |

### Phase 4 — Verify（検証）

- テスト実行（`Test_Encoding.ps1` で文字コード・構文チェック）
- ログファイルのエンコーディングを BOM バイト確認
- CI 通過確認

### Phase 5 — Document（ドキュメント更新）

- `docs/変更履歴.md` に変更を記録
- `CLAUDE.md` の該当セクションを更新
- 関数追加時は `docs/関数リファレンス.md` を更新

## 文字コード強制規定

```powershell
# 必須パターン（全 PS ファイルに適用）
$psver       = $PSVersionTable.PSVersion.Major
$isPS7Plus   = ($psver -ge 7)
$logEncoding = if ($isPS7Plus) { 'utf8NoBOM' } else { 'UTF8' }

# ファイル書き込み時は必ず $logEncoding を使う
Add-Content -Path $path -Value $msg -Encoding $logEncoding
Out-File    -FilePath $path -Append -Encoding $logEncoding
Set-Content -Path $path -Value $content -Encoding $logEncoding
```

詳細は [文字コード規約.md](文字コード規約.md) を参照。

## エラーハンドリング規定

```powershell
# 禁止: 例外を握り潰す
try { ... } catch { }

# 禁止: 全ての操作に SilentlyContinue
Get-Item "path" -ErrorAction SilentlyContinue

# 推奨: ファイル削除など「失敗して当然」な操作のみ SilentlyContinue
Remove-Item "path" -Force -ErrorAction SilentlyContinue

# 推奨: 重要処理はログに記録してから継続
try {
    # 処理
} catch {
    Write-ErrorLog "タスク名 : $_"
    # 処理継続（throw しない）
}
```

## 無限修復防止

- 同一エラーが 3 回繰り返された場合は修復ループを停止
- 状態をユーザーに報告し、手動対応を求める
- 修復ループの上限は 15 回

---

## コード品質チェックリスト

実装完了時に以下をセルフチェックする：

- [ ] 全 `Out-File` / `Add-Content` / `Set-Content` に `-Encoding` が付いている
- [ ] `$logEncoding` が関数定義より前に定義されている
- [ ] 新規関数は `docs/関数リファレンス.md` に追記済み
- [ ] `docs/変更履歴.md` に変更を記録済み
- [ ] PS5.1 と PS7.x の両方で動作することを確認済み
- [ ] 管理者権限が必要な処理に権限チェックが入っている
- [ ] ログにタイムスタンプが付いている

---

## 最重要

**デフォルト禁止。明示・検証・再現性。**

コードの動作は実行環境に依存させない。
すべての前提条件を明示し、コードで検証する。
