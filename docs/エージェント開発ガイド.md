# エージェント開発ガイド

## 1. 概要（Agent DAG の役割）

PC Optimizer の Agent DAG（有向非巡回グラフ）は、最適化処理をプラグイン単位の「エージェント」として分解し、依存関係に従って順次実行するフレームワークです。各エージェントは **collector → analyzer → remediator → reporter** の 4 ロールのうち 1 つ以上を担い、DAG のノードとして `config/config.json` に登録されます。

DAG 実行エンジン（`Orchestration.psm1`）は各ノードを解決し、`Invoke-AgentPlugin` 関数を呼び出してその結果を次のエージェントに引き渡します。

---

## 2. 必要なファイル

カスタムエージェントを実装するには以下のファイルが必要です。

| ファイル | 説明 |
|---|---|
| `modules/agents/YourAgent.psm1` | エージェント本体。`Invoke-AgentPlugin` を公開する |
| `modules/agents/AgentSDK.psm1` | SDK（`New-AgentPluginResult` 等）を提供。Import 必須 |

### `Invoke-AgentPlugin` 関数のシグネチャ

```powershell
function Invoke-AgentPlugin {
    [CmdletBinding()]
    param(
        [string]$Role,         # "collector" | "analyzer" | "remediator" | "reporter"
        [string]$AgentId,      # DAG ノードの agentId
        [pscustomobject]$Node, # config.json の dagNodes エントリ
        [hashtable]$Context    # 実行コンテキスト（RunId, ModuleSnapshot, CollectorByAgent …）
    )
    # このエージェントが担当しない AgentId なら $null を返す
    if ($AgentId -ne "YourAgent") { return $null }
    # ... ロール別処理 ...
}
```

**戻り値:**
- 担当する処理の場合 → `New-AgentPluginResult` で生成したオブジェクト
- 担当しない AgentId の場合 → `$null`（エンジンが次のエージェントにフォールバック）

## 3. ロール別実装例

### collector（データ収集）

```powershell
if ($Role -eq "collector") {
    $payload = [PSCustomObject]@{
        collectedAt = (Get-Date).ToString("s")
        data        = Get-SomeData
    }
    return New-AgentPluginResult -Status Success -Risk Low -Payload $payload -Message "collected"
}
```

### analyzer（分析・スコアリング）

```powershell
if ($Role -eq "analyzer") {
    $col = Get-AgentCollectorResult -Context $Context -AgentId "YourAgent"
    if (-not $col) {
        return New-AgentPluginResult -Status Failed -Risk Unknown -Message "missing-collector"
    }
    $risk = if ($col.payload.data -lt 50) { "High" } else { "Low" }
    return New-AgentPluginResult -Status Success -Risk $risk -Message "analyzed"
}
```

### remediator（修復実行）

```powershell
if ($Role -eq "remediator") {
    # WhatIf モードを尊重する
    if ($Context.WhatIf) {
        return New-AgentPluginResult -Status Skipped -Risk Low -Message "whatif-skip"
    }
    Invoke-Remediation
    return New-AgentPluginResult -Status Success -Risk Low -Message "remediated"
}
```

### reporter（レポート生成）

```powershell
if ($Role -eq "reporter") {
    $summary = [PSCustomObject]@{ agentId = $AgentId; status = "ok" }
    return New-AgentPluginResult -Status Success -Risk Low -Payload $summary -Message "reported"
}
```

## 4. SampleAgent.Template.psm1 のコード解説

`modules/agents/SampleAgent.Template.psm1` は最小構成のエージェントテンプレートです。

```powershell
$sdkPath = Join-Path $PSScriptRoot "AgentSDK.psm1"
Import-Module $sdkPath -Force -ErrorAction Stop
```

AgentSDK を必ず `Import-Module` で読み込みます。`-ErrorAction Stop` により SDK が見つからない場合は即座にエラーになります。

```powershell
if (-not (Test-AgentPluginContext -Context $Context)) {
    return New-AgentPluginResult -Status Failed -Risk Unknown -Message "invalid-context"
}
if ("$AgentId" -ne "SampleAgent") { return $null }
```

コンテキスト検証（`RunId` と `ModuleSnapshot` の存在確認）を先頭で行います。`AgentId` が自身でなければ `$null` を返して処理をスキップします。

**collector ロール:** `$Context.RunId` を埋め込んだペイロードを生成して返します。

**analyzer ロール:** `Get-AgentCollectorResult` で同一エージェントのコレクター結果を取得し、存在しなければ `Failed` を返します。

## 5. 登録方法（config.json の dagNodes への追加）

`config/config.json` の `dagNodes` 配列に以下の形式でノードを追加します。

```json
{
  "dagNodes": [
    {
      "agentId": "YourAgent",
      "modulePath": "modules/agents/YourAgent.psm1",
      "roles": ["collector", "analyzer"],
      "depends": [],
      "enabled": true
    }
  ]
}
```

| フィールド | 説明 |
|---|---|
| `agentId` | エージェントの一意な識別子 |
| `modulePath` | リポジトリルートからの相対パス |
| `roles` | 実行するロールの配列 |
| `depends` | 依存する agentId の配列（空なら先行制約なし） |
| `enabled` | `false` にするとスキップ |

## 6. テスト方法（AgentSDK.Tests.ps1 への追加）

`tests/AgentSDK.Tests.ps1` にカスタムフレームワーク（`Assert-True`）形式でテストを追記してください。

```powershell
# モジュール読み込み
Import-Module (Join-Path $RepoRoot "modules\agents\YourAgent.psm1") -Force

# 有効なコンテキストを用意
$ctx = @{
    RunId            = "test-run"
    ModuleSnapshot   = [PSCustomObject]@{}
    CollectorByAgent = @{}
}

# collector ロールのテスト
$col = Invoke-AgentPlugin -Role "collector" -AgentId "YourAgent" -Node $null -Context $ctx
Assert-True "YA-01: YourAgent collector returns Success" ($col.status -eq "Success")

# analyzer ロール（collector 結果を Context に注入）
$ctx.CollectorByAgent["YourAgent"] = $col
$ana = Invoke-AgentPlugin -Role "analyzer" -AgentId "YourAgent" -Node $null -Context $ctx
Assert-True "YA-02: YourAgent analyzer returns Success" ($ana.status -eq "Success")
```

テストの実行:
```powershell
cd D:\PC_Optimizer
powershell -NoProfile -ExecutionPolicy Bypass -File tests\AgentSDK.Tests.ps1
```

Pester テストとして追加する場合は `tests/AgentTeams.Pester.Tests.ps1` か `tests/PCOptimizer.Pester.Tests.ps1` の末尾に `Describe "Module: YourAgent.psm1"` ブロックを追記してください。
