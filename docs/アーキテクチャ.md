# アーキテクチャ

## 概要

2 ファイルで構成されるシンプルなモノリシックスクリプトです。
依存ライブラリはなく、Windows 標準の PowerShell と組み込みコマンドのみで動作します。

```
Run_PC_Optimizer.bat
  │  管理者昇格 + PowerShell 検出
  └─► PC_Optimizer.ps1
        │  ログ初期化（logs/ フォルダに出力）
        │  バージョン検出
        │  システム情報収集・表示
        │  初期ディスク空き容量記録
        │  最適化タスク × 20（Try-Step でラップ）
        │  ディスク解放量の比較表示
        └─► 再起動チェック → 終了
```

## ファイル構成

| ファイル / フォルダ | 役割 |
|---|---|
| `PC_Optimizer.ps1` | 最適化エンジン本体 |
| `Run_PC_Optimizer.bat` | 管理者昇格 + PowerShell 検出ランチャー |
| `logs/` | 実行ログの出力先（自動生成） |
| `docs/` | ドキュメント群 |

## BAT ランチャーの処理フロー

```
Run_PC_Optimizer.bat
  1. pwsh.exe (PS7) を where コマンドで検索
  2. 見つかれば pwsh.exe を使用、なければ powershell.exe (PS5.1) を使用
  3. net session で管理者権限を確認
  4. 権限なし → Start-Process -Verb RunAs で自身を管理者再起動
  5. 権限あり → -ExecutionPolicy Bypass で PS1 を実行
```

## PowerShell スクリプトの処理フロー

### 1. 初期化フェーズ

```powershell
$now          = Get-Date -Format "yyyyMMddHHmm"
$logsDir      = Join-Path $PSScriptRoot "logs"   # logs/ フォルダを自動作成
$logPath      = ...   # logs\PC_Optimizer_Log_YYYYMMDDHHMM.txt
$errorLogPath = ...   # logs\PC_Optimizer_Error_YYYYMMDDHHMM.txt
$psver        = $PSVersionTable.PSVersion.Major
$isPS7Plus    = ($psver -ge 7)
```

### 2. 情報収集フェーズ

`Get-CimInstance Win32_*` で CPU・メモリ・ディスク情報を取得し、
`$isPS7Plus` フラグに応じて絵文字（PS7+）または ASCII 記号（PS5.x 以下）で表示します。

```
$isPS7Plus = true  → 🖥️ 👤 🏷️ 💻 🧠 💾 の絵文字付き表示
$isPS7Plus = false → * 記号の ASCII 表示
```

### 3. タスク実行フェーズ（20 タスク）

すべての最適化処理は `Try-Step` ラッパー経由で実行されます。

```
Try-Step "タスク名" {
    # 実際の処理
}
  ↓
  開始ログ書き込み
  Progress-Bar 表示（0%）
  try { ScriptBlock 実行 }
  catch { エラーログ書き込み }  ← 失敗しても次タスクへ継続
  Progress-Bar 表示（100%）
  完了/失敗ログ書き込み
```

**重要**: `Try-Step` は各タスクを独立して実行します。
1 つのタスクが失敗しても後続タスクは継続されます（非ブロッキング設計）。

### 4. 後処理フェーズ

```
ディスク解放量の比較表示（$initialFreeGB → $finalFreeGB）
  ↓
Test-PendingReboot でレジストリを確認
  ↓
再起動必要 → ユーザーに Y/N を問う
  ↓
完了メッセージ → Read-Host で終了待機
```

## 主要関数の依存関係

```
Write-Log           ← Add-Content (PS 2.0+)
Write-ErrorLog      ← Add-Content (PS 2.0+)
Show                ← Write-Host (全バージョン)
Progress-Bar        ← Write-Host + $isPS7Plus 分岐
Try-Step            ← Progress-Bar, Write-Log, Write-ErrorLog
Test-PendingReboot  ← Test-Path, Get-ItemProperty (PS 2.0+)
Run-WindowsUpdate   ← Start-Process, UsoClient.exe の存在確認
```

## エラー処理設計

```
ファイル削除系 → -ErrorAction SilentlyContinue（ロック中ファイルをサイレントスキップ）
各タスク全体  → Try-Step の try-catch（タスク失敗時に Write-ErrorLog して継続）
情報収集      → try-catch（失敗しても処理を止めない）
外部コマンド  → $LASTEXITCODE で成否判定（wevtutil, powercfg など）
```

## ログ設計

ログは `logs/` サブフォルダに 2 種類生成されます。

```
logs/PC_Optimizer_Log_YYYYMMDDHHMM.txt      ← 通常ログ
logs/PC_Optimizer_Error_YYYYMMDDHHMM.txt    ← エラーログ（エラー発生時のみ）
```

詳細は [ログ仕様.md](ログ仕様.md) を参照してください。

## 拡張ポイント

新しい最適化タスクを追加する場合は、
`# ==========================` セクション内に `Try-Step "タスク名" { ... }` の形式で追加します。

```powershell
Try-Step "新しいタスク" {
    # 処理内容
}
```
