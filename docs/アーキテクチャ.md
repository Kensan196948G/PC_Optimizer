# アーキテクチャ

## 概要

モジュール分割構成のスクリプトツールです。
依存ライブラリはなく、Windows 標準の PowerShell と組み込みコマンドのみで動作します（外部 API はオプション）。

```
Run_PC_Optimizer.bat
  │  管理者昇格 + PowerShell 検出
  └─► PC_Optimizer.ps1
        │  モジュールロード（modules/*.psm1）
        │  ログ初期化（logs/ フォルダに出力）
        │  バージョン検出
        │  システム情報収集・表示
        │  初期ディスク空き容量記録
        │  最適化タスク × 20（Invoke-GuardedStep でラップ）
        │  ディスク解放量の比較表示
        │  レポート生成（reports/ フォルダ）
        │  Agent Teams DAG 実行（Orchestration.psm1）
        └─► 再起動チェック → 終了
```

## BAT ランチャーの処理フロー

```
Run_PC_Optimizer.bat
  1. pwsh.exe (PS7) を where コマンドで検索
  2. 見つかれば pwsh.exe を使用、なければ powershell.exe (PS5.1) を使用
  3. net session で管理者権限を確認
  4. 権限なし → Start-Process -Verb RunAs で自身を管理者再起動
  5. 権限あり → -ExecutionPolicy Bypass で PS1 を実行
```

## PowerShell スクリプトの処理フロー

### 1. 初期化フェーズ

```powershell
$now          = Get-Date -Format "yyyyMMddHHmm"
$logsDir      = Join-Path $PSScriptRoot "logs"   # logs/ フォルダを自動作成
$logPath      = ...   # logs\PC_Optimizer_Log_YYYYMMDDHHMM.txt
$errorLogPath = ...   # logs\PC_Optimizer_Error_YYYYMMDDHHMM.txt
$psver        = $PSVersionTable.PSVersion.Major
$isPS7Plus    = ($psver -ge 7)
# modules/ を Import-Module でロード
```

### 2. 情報収集フェーズ

`Get-CimInstance Win32_*` で CPU・メモリ・ディスク情報を取得し、
`$isPS7Plus` フラグに応じて絵文字（PS7+）または ASCII 記号（PS5.x 以下）で表示します。

```
$isPS7Plus = true  → 🖥️ 👤 🏷️ 💻 🧠 💾 の絵文字付き表示
$isPS7Plus = false → * 記号の ASCII 表示
```

### 3. タスク実行フェーズ（20 タスク）

すべての最適化処理は `Invoke-GuardedStep` ラッパー経由で実行されます。

```
Invoke-GuardedStep "タスク名" {
    # 実際の処理
}
  ↓
  開始ログ書き込み
  Progress-Bar 表示（0%）
  try { ScriptBlock 実行 }
  catch { エラーログ書き込み }  ← 失敗しても次タスクへ継続
  Progress-Bar 表示（100%）
  完了/失敗ログ書き込み
```

**重要**: `Invoke-GuardedStep` は各タスクを独立して実行します。
1 つのタスクが失敗しても後続タスクは継続されます（非ブロッキング設計）。

### 4. レポート・DAG フェーズ

```
Report.psm1 でレポート生成（reports/ フォルダ）
  ↓
Orchestration.psm1 → Agent Teams DAG 実行（下記参照）
  ↓
ディスク解放量の比較表示（$initialFreeGB → $finalFreeGB）
  ↓
Test-PendingReboot でレジストリを確認
  ↓
再起動必要 → ユーザーに Y/N を問う
  ↓
完了メッセージ → Read-Host で終了待機
```

## Agent Teams DAG（Orchestration.psm1）

```
planner
  → [collector.security, collector.network, collector.update]  ← 並行実行
    → analyzer.*
      → analyzer.aggregate
        → remediator
          → reporter
            → on_report フック自動発火
            → Invoke-McpProviders（Slack/Teams/ServiceNow/Jira dispatch）
```

## Hook / MCP / SIEM 設計

### Hook タイプ（Orchestration.psm1）

| タイプ | 動作 | 備考 |
|---|---|---|

## エラー処理設計

```
ファイル削除系  → -ErrorAction SilentlyContinue（ロック中ファイルをサイレントスキップ）
各タスク全体   → Invoke-GuardedStep の try-catch（タスク失敗時に Write-ErrorLog して継続）
情報収集       → try-catch（失敗しても処理を止めない）
外部コマンド   → $LASTEXITCODE で成否判定（wevtutil, powercfg など）
$null 比較     → $null -ne $x の順序に統一（v4.0.1 修正済み）
```

## セキュリティ設計（v4.0.1 修正）

| 項目 | 内容 |
|---|---|
| `Invoke-Expression` 廃止 | command 型 Hook を削除し、コードインジェクションリスクを排除 |
| コマンドインジェクション修正 | `cmd.exe` 経由削除を `Remove-Item -LiteralPath` に変更 |
| `$null` 比較順序統一 | `$null -ne $x` 形式に統一（PowerShell の型強制バグ回避） |
| エンコーディング統一 | `$script:_enc` 変数で全モジュールのエンコーディングを一元管理 |
| Hook タイプ制限 | `webhook` / `file` タイプのみ許可（`command` タイプは廃止） |

## ログ設計

ログは `logs/` サブフォルダに 3 種類生成されます。

```
logs/PC_Optimizer_Log_YYYYMMDDHHMM.txt      ← 通常ログ
logs/PC_Optimizer_Error_YYYYMMDDHHMM.txt    ← エラーログ（エラー発生時のみ）
logs/siem/                                  ← SIEM ログ（hooks.siem 有効時のみ）
```

詳細は [ログ仕様.md](ログ仕様.md) を参照してください。

## 拡張ポイント

### 新しい最適化タスクを追加する場合

`# ==========================` セクション内に `Invoke-GuardedStep "タスク名" { ... }` の形式で追加します。

```powershell
Invoke-GuardedStep "新しいタスク" {
    # 処理内容
}
```

### 新しいモジュールを追加する場合

1. `modules/NewModule.psm1` を作成し、`Common.psm1` を Import する
2. `PC_Optimizer.ps1` 冒頭の `Import-Module` リストに追加する
3. `config/` に設定スキーマを追加する（必要な場合）

### カスタムエージェントを追加する場合

`modules/agents/` フォルダにエージェントスクリプトを配置し、
`Orchestration.psm1` の DAG 定義に登録します。
