# v4.0 移行計画（v3.3.x -> v4.0）

作成日: 2026-03-04

## 1. 前提
- 現行安定版: v3.3.1
- 移行先: v4.0.0
- 対象: 社内IT部門運用端末

## 2. 段階移行
- Phase 1: 設計固定
  - CLI仕様、JSONスキーマ、終了コードを先に確定
- Phase 2: 並行実装
  - 既存処理を維持しつつ modules 化
- Phase 3: 互換検証
  - PS5.1/PS7 の同等結果を確認
- Phase 4: 本番切替
  - `release/v4.0.0` でRC実施後、タグ確定

## 3. 互換ポリシー
- 既存の引数なし実行は維持
- `Run_PC_Optimizer.bat` は追加CLIを透過転送
- 既存ログフォーマットは後方互換を優先

## 4. リスクと対策
- リスク: モジュール分割で既存動作が変わる
  - 対策: スナップショットテストと差分比較
- リスク: WhatIf の抜け漏れ
  - 対策: 変更系関数を共通ラッパー経由に統一
- リスク: 運用者の手順混在
  - 対策: README と運用手順をv4.0準拠へ更新

## 5. 切替完了条件
- v4.0 の必須機能（診断、更新、レポート）完了
- CI 2系統（5.1/7）安定稼働
- リリースノートと運用手順の更新完了
