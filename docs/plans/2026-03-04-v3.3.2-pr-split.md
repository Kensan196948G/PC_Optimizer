# v3.3.2 実装PR分割計画

## 方針
- 変更影響を局所化するため、`-NonInteractive` と `-WhatIf` を別PRで実装する。
- 先に契約（CLI仕様・JSONスキーマ）を固定してから実装に入る。

## PR-A: NonInteractive 基盤
- ブランチ: `feature/v3.3.2-noninteractive`
- 目的:
  - `-NonInteractive` と `-NoRebootPrompt` の導入
  - 対話プロンプトのポリシー化（既定値の明示）
- 主な変更:
  - 引数パース拡張
  - 再起動・更新確認の非対話分岐
  - 終了コード標準化（自動化向け）
- 受け入れ:
  - 非対話実行で停止しない
  - 既存対話動作は維持

## PR-B: WhatIf / Export
- ブランチ: `feature/v3.3.2-whatif`
- 目的:
  - `-WhatIf` と `-ExportDeletedPaths` の導入
  - 削除系処理のプレビュー出力
- 主な変更:
  - 変更系処理を抽象化し、実行/プレビューを切替
  - CSV/JSON の削除予定パス出力
  - HTML/JSON レポートへ preview フラグを追加
- 受け入れ:
  - `-WhatIf` でシステム変更なし
  - 削除予定一覧が出力される

## PR-C: JSONレポート / 選択実行（後続）
- ブランチ: `feature/v3.3.2-json-report`
- 目的:
  - `-OutputFormat json|both`
  - `-Tasks` / `-FailureMode` 対応
