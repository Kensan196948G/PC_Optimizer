# v3.3.2 CLI拡張 仕様骨子

作成日: 2026-03-04  
対象ブランチ: `feature/v3.3.2-foundation`

## 1. 目的
- 対話実行専用だった `PC_Optimizer.ps1` を自動運用しやすいCLIへ拡張する。
- 互換性（PS5.1/PS7）を維持しつつ、CI連携可能な出力形式を標準化する。

## 2. 追加CLI引数（v3.3.2候補）
- `-NonInteractive`
  - Y/Nプロンプトを自動応答する。
  - 未指定時は現行どおり対話動作。
- `-WhatIf`
  - 実際の変更を行わず、実行予定タスクと削除対象を表示/記録する。
- `-Tasks <string>`
  - 例: `-Tasks "1,3,7"` で対象タスクのみ実行。
- `-LogLevel <info|warn|error|debug>`
  - ログ出力粒度を制御。
- `-NoRebootPrompt`
  - 再起動確認プロンプトを抑制（自動化向け）。
- `-OutputFormat <html|json|both>`
  - レポート出力形式を選択。
- `-ExportDeletedPaths <csv|json>`
  - 削除予定パス一覧を事前エクスポート。
- `-FailureMode <continue|fail-fast>`
  - タスク失敗時の継続/即時停止を切替。

## 3. 実行モード設計
- Interactiveモード（既定）
  - 現行と同じ対話式。
- Automationモード（`-NonInteractive`）
  - 対話を禁止し、事前定義ポリシーで自動判断。
- Previewモード（`-WhatIf`）
  - 変更系コマンドを無効化し、ログ/レポートのみ出力。
- Partialモード（`-Tasks`）
  - 指定タスク集合のみ実行。

## 4. 互換性方針
- PowerShell 5.1 / 7.x の両対応を必須。
- 既存タスク定義・既存ログフォーマットを破壊しない（後方互換）。
- 既存 `Run_PC_Optimizer.bat` は追加引数を透過的に渡せる設計へ拡張。

## 5. 受け入れ基準（骨子）
- 既存テストが全PASS。
- 新規モードの基本動作テストを追加。
- 非対話環境でハングしない。
- JSON出力がスキーマ検証を通過。
