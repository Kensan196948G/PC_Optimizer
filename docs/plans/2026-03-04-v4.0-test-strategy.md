# v4.0 テスト戦略

作成日: 2026-03-04  
テスト基盤: Pester + GitHub Actions

## 1. テスト層
- Unit
  - モジュール関数単位（Verb-Noun）
- Integration
  - `PC_Optimizer.ps1` のモード別実行
- E2E
  - 管理者権限前提での実運用シナリオ

## 2. 重点検証
- `-NonInteractive` で入力待ちゼロ
- `-WhatIf` で副作用ゼロ
- `-Tasks` 指定時の実行対象制御
- `-FailureMode` の挙動差分
- JSON/CSV/HTML 出力の整合

## 3. 互換性検証
- Windows PowerShell 5.1
- PowerShell 7.x
- 同じテストセットを両ランタイムで実行

## 4. CI要件
- `.github/workflows/ci.yml`
- 必須ジョブ:
  - `Test (Windows PowerShell 5.1)`
  - `Test (PowerShell 7)`
- artifact:
  - `logs/*.log`
  - `reports/*.html`, `reports/*.json`, `reports/*.csv`
  - retention: 14日

## 5. リリース判定（Go/No-Go）
- 必須テスト全PASS
- 既知制約をリリースノートへ記載
- 破壊的変更は移行手順に明記
