# v4.0 再設計アーキテクチャ

作成日: 2026-03-04  
版: 4.0  
対象: 社内IT部門向け PC_Optimizer

## 1. 設計方針
- 既存の「一括実行スクリプト」を、責務単位モジュールへ分割する。
- 非対話運用を標準化し、CI/定期実行（Task Scheduler）に適合させる。
- 出力契約を先に固定し、HTML/CSV/JSON を同じ実行結果モデルから生成する。
- PowerShell 5.1 / 7.x の互換性を維持する。

## 2. 目標ディレクトリ構成
```text
PC_Optimizer
├─ PC_Optimizer.ps1
├─ Run_PC_Optimizer.bat
├─ config
│  └─ config.json
├─ modules
│  ├─ Cleanup.psm1
│  ├─ Diagnostics.psm1
│  ├─ Performance.psm1
│  ├─ Network.psm1
│  ├─ Security.psm1
│  ├─ Update.psm1
│  └─ Report.psm1
├─ reports
├─ logs
├─ docs
├─ tests
└─ .github/workflows
```

## 3. 実行レイヤ
- Entry Layer
  - `Run_PC_Optimizer.bat`
  - `PC_Optimizer.ps1`（引数解析、構成読込、終了コード制御）
- Domain Layer
  - `modules/*.psm1`（診断、最適化、更新、セキュリティ）
- Report Layer
  - 実行結果オブジェクトから HTML/CSV/JSON を生成
- Infra Layer
  - ログ出力、環境判定、管理者権限判定、例外処理

## 4. 機能マッピング（要件 10〜24）
- Diagnostics.psm1
  - システム診断、IT資産情報、イベントログ分析
- Performance.psm1
  - CPU/メモリ/ディスク分析、スタートアップ分析、スコア計算
- Security.psm1
  - Defender/Firewall/BitLocker/UAC 診断
- Network.psm1
  - IP/DNS/Gateway/NIC 速度
- Update.psm1
  - Windows Update 状態、更新履歴、更新エラー、Microsoft 365 診断
- Cleanup.psm1
  - Temp/キャッシュ削除、DNS修復などの自動修復タスク
- Report.psm1
  - PC Health Report、改善提案、Chart.js 用データ

## 5. 実行モード
- Interactive
  - 従来互換の対話モード
- NonInteractive
  - 自動運用向け（確認をポリシーで処理）
- WhatIf/DryRun
  - 変更なしの事前評価
- Remote（将来）
  - WinRM/Remoting による複数PC診断

## 6. セキュリティ設計
- 管理者権限の強制確認
- 削除対象パスを allow-list 管理
- `.env` を Git 管理対象外
- 例外時も `finally` で監査ログを確実に記録

## 7. 互換性設計
- 5.1/7.x 共通で使えるコマンドを優先
- バージョン差分が出る箇所は `compat` 関数に隔離
- CI で両ランタイムを必須チェック化
