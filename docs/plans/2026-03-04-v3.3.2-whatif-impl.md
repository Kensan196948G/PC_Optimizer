# v3.3.2 実装計画: -WhatIf / -ExportDeletedPaths

作成日: 2026-03-04  
対象ブランチ: `feature/v3.3.2-whatif`

## 1. スコープ
- `-WhatIf` を追加し、変更処理を実行せずに影響範囲を可視化する。
- `-ExportDeletedPaths` を追加し、削除対象一覧を `csv` または `json` で出力する。
- JSONレポートに preview 実行情報を載せるための最小契約を固定する。

## 2. CLI仕様（初版）
- `-WhatIf`
  - 指定時: 削除・更新・設定変更コマンドは実行しない。
  - 出力: 実行予定タスク、想定対象件数、対象パス一覧（可能な範囲）。
- `-ExportDeletedPaths <csv|json>`
  - 指定時: 削除候補をファイル出力する。
  - `-WhatIf` との併用を基本とする。

## 3. 実装方針
- 変更系処理をラップして「実行/プレビュー」を切替できる構造へ寄せる。
- プレビュー時は副作用を持つ呼び出しを禁止する。
- 既存ログに `preview=true/false` を追加可能な拡張点を用意する。

## 4. レポート契約（初版）
- トップレベル:
  - `status`, `durationSeconds`, `tasks`
- タスク単位:
  - `id`, `name`, `status`, `duration`, `errors`, `previewOnly`
- スキーマは `docs/schemas/pc-optimizer-report-v1.schema.json` を参照する。

## 5. 受け入れ基準
- `-WhatIf` 実行でシステム変更が発生しない。
- 削除候補一覧が `csv/json` で出力される。
- 既存テストが退行しない。
