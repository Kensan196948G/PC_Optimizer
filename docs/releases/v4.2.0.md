# v4.2.0 リリースノート

**リリース日**: 2026-03-05
**タグ**: `v4.2.0`

## 概要

v4.1.0 の通知機能を拡張。ServiceNow/Jira 自動起票・HTML レポート印刷対応・SIEM 出力統合・on_report フック自動発火を追加。

## 主な変更

### 追加

- **`modules/Notification.psm1`** ServiceNow/Jira 起票追加
  - `Send-ServiceNowIncident`: スコア閾値未満の場合に ServiceNow インシデントを自動起票（urgency 自動設定）
  - `Send-JiraTask`: スコア閾値未満の場合に Jira タスクを自動起票（Basic 認証対応）

- **`modules/Orchestration.psm1`** SIEM 統合・on_report フック
  - `Invoke-AgentTeamsOrchestration` の末尾で `on_report` フックを自動発火
  - Slack/Teams 通知をリッチカード形式に強化（スコア・評価・推奨アクション・ホスト名）
  - ServiceNow/Jira にスコア閾値チェックを追加（scoreThreshold: 70 以下のとき起票）
  - `Convert-HookEntryToSiemLine` / `Export-HookSiemLines` を Export-ModuleMember に追加

- **`modules/Report.psm1`** 印刷・エクスポート強化
  - `@media print` CSS 追加（印刷時ボタン非表示・背景除去）
  - メタ情報バー追加（PC名・実行フォルダ・生成日時）
  - 「🖨️ 印刷/PDF保存」ボタン追加

- **`tests/Test_SiemOutput.ps1`** 新規作成 — SI-01〜SI-08（24 PASS）

- **`config/config.json`** 設定強化
  - `hooks.on_report` に `audit-report-log` エントリを追加
  - `mcpProviders.servicenow-ticket` / `jira-ticket` に `scoreThreshold: 70` を追加

### セキュリティ（v4.0.1 修正内容）

- Hook の `command` 型で `Invoke-Expression` を使用していた脆弱性を修正
- `cmd.exe` + ダブルクォート引数によるコマンドインジェクションを修正
- エンコーディングを `$script:_enc` 変数に統一

## オプション機能（デフォルト無効）

| 機能 | 有効化方法 |
|---|---|
| Slack 通知 | mcpProviders[].type="slack", enabled=true, webhookUrl 設定 |
| Teams 通知 | mcpProviders[].type="teams", enabled=true, webhookUrl 設定 |
| ServiceNow 起票 | mcpProviders[].type="servicenow", enabled=true, scoreThreshold 設定 |
| Jira 起票 | mcpProviders[].type="jira", enabled=true, scoreThreshold 設定 |
| SIEM 出力 | hooks.siem.enabled=true |

詳細は [config仕様.md](../config仕様.md) を参照してください。

## 動作確認済み環境

- Windows 11 24H2 / PowerShell 7.4.x
- Windows 10 21H2 / PowerShell 5.1
