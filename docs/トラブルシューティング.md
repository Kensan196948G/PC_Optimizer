# トラブルシューティング

## よくある問題

### Q1: 「管理者として実行」しても処理がすぐ終わる

**原因**: PowerShell の実行ポリシーでスクリプトがブロックされている。

**対処**:

```powershell
# 管理者権限の PowerShell で実行
Set-ExecutionPolicy -Scope LocalMachine -ExecutionPolicy RemoteSigned
```

または、BAT ファイルを使用してください（`-ExecutionPolicy Bypass` が自動付与されます）。

---

### Q2: 一部のファイルが削除できない

**原因**: 実行中のアプリケーションがファイルをロックしている。

**対処**:
1. 不要なアプリケーション（特に Office, Teams, OneDrive, Chrome）を閉じてから再実行
2. それでも削除できない場合は再起動後に再実行
3. エラーログ（`logs\PC_Optimizer_Error_*.txt`）で詳細を確認

削除できないファイルは `-ErrorAction SilentlyContinue` でスキップされるため、
他の処理には影響しません。

---

### Q3: 日本語が文字化けする

**原因**: スクリプトファイルのエンコーディングが不正。

**対処**: `PC_Optimizer.ps1` を **UTF-8 with BOM** で保存し直してください。

```powershell
# PowerShell 7 で変換する場合
$content = Get-Content "PC_Optimizer.ps1" -Raw
Set-Content "PC_Optimizer.ps1" -Value $content -Encoding utf8BOM
```

VS Code の場合: 右下のエンコーディング表示 → `Save with Encoding` → `UTF-8 with BOM`

---

### Q4: ディスク最適化でエラーが出る

**原因 1**: ディスクに問題がある（不良セクタなど）。

**対処**: `chkdsk C: /f` でディスクエラーを修復してから再実行。

**原因 2**: `Get-PhysicalDisk` が利用できない環境。

**対処**: エラーログを確認。Storage モジュールの問題であればスクリプトは HDD モードでフォールバックします。

---

### Q5: Windows Update が実行されない

**原因 1**: `UsoClient.exe` が見つからない（Windows 10 以前の環境）。

**対処**: Windows 10 / 11 のみ対応です。Windows 8.1 以前では Windows Update タスクはスキップされます。

**原因 2**: Windows Update サービスが停止している。

**対処**:

```powershell
# Update サービスの状態確認
Get-Service -Name wuauserv, UsoSvc

# サービスを起動
Start-Service -Name wuauserv
Start-Service -Name UsoSvc
```

---

### Q6: 「警告: PowerShell 3.0 以上を推奨します。」と表示される

**原因**: PowerShell 2.0 以下で実行している。

**対処**: PowerShell 5.1 または 7.x を使用してください。
Windows 10/11 では標準搭載の PowerShell 5.1 で問題なく動作します。

---

### Q7: SSD 健康診断で「SSD が検出されませんでした。」と表示される

**原因 1**: SATA/USB 接続の SSD を使用しており、`MediaType` が正しく取得できない場合があります。

**確認方法**:

```powershell
Get-PhysicalDisk | Select-Object FriendlyName, MediaType, OperationalStatus
```

`MediaType` が `Unspecified` になっている場合、ドライバーまたは接続方式の問題です。

**原因 2**: M.2 NVMe SSD が別の `DeviceID` を持っている場合。
スクリプトは `$_.MediaType -eq "SSD"` でフィルタリングしているため、
`MediaType` が正しく設定されていれば検出されます。

---

### Q8: スクリプト実行後にログファイルが増えていく

ログは `logs\` サブフォルダに実行のたびに新しいファイルが生成されます（自動ローテーションなし）。
不要なログを定期的に削除してください。

```powershell
# logs フォルダ内の古いログを削除（30日以上前）
Get-ChildItem -Path "$PSScriptRoot\logs" -Filter "PC_Optimizer_*.txt" |
    Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-30) } |
    Remove-Item -Force
```

---

### Q9: SFC や DISM で時間がかかる

**これは正常な動作です。**

- SFC (`sfc /scannow`) はシステムファイル全体をスキャン・修復するため 5〜15 分かかる場合があります
- DISM (`/ScanHealth`) はコンポーネントストアの診断のみ実行し、修復は行いません（5〜10 分程度）

両タスクとも完了まで待機します。

---

### Q10: Microsoft 365 が検出されない

**原因 1**: Microsoft 365 が Click-to-Run 形式でインストールされていない（MSI 版など）。

**確認方法**:

```powershell
Test-Path "C:\Program Files\Common Files\Microsoft Shared\ClickToRun\OfficeC2RClient.exe"
Test-Path "C:\Program Files (x86)\Common Files\Microsoft Shared\ClickToRun\OfficeC2RClient.exe"
```

どちらも `False` の場合、Click-to-Run ではないため自動的にスキップされます（正常な動作）。

**原因 2**: Office の永続ライセンス版（Office 2019/2021 など）でも Click-to-Run 形式なら検出可能です。

---

### Q11: Windows Update の確認に時間がかかる

**これは正常な動作です。**

COM API（`Microsoft.Update.Session`）による更新の検索は Windows Update サービスに問い合わせるため、ネットワーク状況によっては数十秒かかります。「しばらくお待ちください」メッセージが表示されている間は処理中です。

---

### Q12: イベントログのクリアが失敗する

**原因**: Security ログは対象外です。Application / System のみクリアされます。

**確認方法**:

```powershell
# イベントログのクリアを手動実行（管理者権限必須）
wevtutil cl Application
wevtutil cl System
```

`$LASTEXITCODE` が `0` 以外の場合はエラーログに記録されます。

---

### Q13: AI 診断の結果が文字化けする

**原因**: PowerShell 5.1 で AI 診断 API のレスポンスが ANSI エンコーディングで誤デコードされていた（v4.0.1 で修正済み）。

**確認方法**: バージョンが v4.0.1 以降であることを確認してください。

**最新版へのアップデート**: ツールフォルダを最新バージョンに差し替えてください。

---

### Q14: Slack/Teams への通知が届かない

**確認事項**:
1. `config.json` の `mcpProviders` セクションで `"enabled": true` になっているか確認
2. `webhookUrl` が正しく設定されているか確認（Slack は `https://hooks.slack.com/...`、Teams は `https://*.webhook.office.com/...`）
3. 対象 PC からインターネット接続できるか確認

**エラーログの確認**:
```
logs\PC_Optimizer_Error_*.txt に Notification エラーが記録されます。
```

---

### Q15: ServiceNow / Jira へのチケットが起票されない

**原因 1**: スコアが `scoreThreshold`（デフォルト 70）以上の場合は起票されません（正常動作）。

**原因 2**: `enabled` が `false` になっている。

**確認方法**:
```powershell
# config.json の設定を確認
Get-Content .\config\config.json | ConvertFrom-Json | Select-Object -ExpandProperty mcpProviders
```

---

### Q16: SIEM ファイルが出力されない

**原因**: `config.json` で `hooks.siem.enabled` が `false`（デフォルト）になっている。

**有効化方法**:
```json
"hooks": {
  "siem": {
    "enabled": true,
    "format": "jsonl",
    "outputPath": "logs\\siem"
  }
}
```

---

### Q17: Agent Teams モードが実行されない

**原因**: `config.json` で `agentTeams.enabled` が `false` になっている、または AI 診断が無効。

**有効化方法**:
```json
{
  "agentTeams": { "enabled": true },
  "aiDiagnostics": { "enabled": true, "apiKey": "sk-ant-..." }
}
```

---

### Q18: PS7 で「StrictMode」関連の警告が出る

**これは v4.0.1 以前のバグで、v4.0.1 で修正済みです。**

修正内容: `$provider.retryCount` 等のプロパティアクセス前に `PSObject.Properties` ガードを追加。

最新版では警告は発生しません。古いバージョンを使用している場合はアップデートしてください。

## エラーログの読み方

```
[ERROR] 2026/02/19 12:00:15
一時ファイルの削除 : パス '...' へのアクセスが拒否されました。
```

- 1 行目: エラー発生タイムスタンプ
- 2 行目: `タスク名 : エラーメッセージ`

一般的なエラーメッセージ：

| メッセージ | 意味 |
|---|---|
| `パス '...' へのアクセスが拒否されました` | ファイルがロック中または権限不足 |
| `パス '...' が見つかりません` | 対象パスが存在しない（通常は無害） |
| `UsoClient.exe が見つかりません` | Windows 10 以前の環境 |
| `Get-PhysicalDisk not available` | Storage モジュール未搭載 |
