# インストール・前提条件

## 前提条件

### OS

- Windows 10 (バージョン 1903 以降推奨)
- Windows 11

### PowerShell

| バージョン | 状態 |
|---|---|
| 5.1 | Windows 10/11 標準搭載。追加インストール不要 |
| 7.x | 任意インストール。絵文字・ANSI カラー表示が有効になる |
| 3.0 / 4.0 | 動作するが一部機能制限あり（後述） |
| 2.0 以下 | 非対応（`Get-CimInstance` 未対応のため） |

PowerShell 7 は [GitHub Releases](https://github.com/PowerShell/PowerShell/releases) からインストール可能です。

### 実行権限

**管理者権限が必須**です。標準ユーザーで実行すると以下の処理が失敗します。

- `C:\Windows\Temp` の削除
- `C:\Windows\SoftwareDistribution` の削除
- ディスク最適化 (`Optimize-Volume` / `defrag`)
- Windows Update の実行

## ファイル配置

本ツールは**対象 PC の任意フォルダにコピーして実行**するスタンドアロンツールです。
フォルダごとコピーして `Run_PC_Optimizer.bat` を管理者権限で実行してください。

```
任意のフォルダ\PC_Optimizer\
├── PC_Optimizer.ps1          ← メインスクリプト
├── Run_PC_Optimizer.bat      ← 起動ランチャー
├── config\
│   └── config.json           ← 設定ファイル（オプション機能の有効化等）
├── modules\
│   ├── Common.psm1
│   ├── Cleanup.psm1
│   ├── Diagnostics.psm1
│   ├── Performance.psm1
│   ├── Security.psm1
│   ├── Network.psm1
│   ├── Update.psm1
│   ├── Report.psm1
│   ├── Advanced.psm1
│   ├── Orchestration.psm1
│   ├── Notification.psm1
│   └── agents\               ← カスタムエージェント
├── tests\                    ← テストスクリプト（開発用）
├── logs\                     ← 実行ログ（自動生成）
├── reports\                  ← レポート出力先（自動生成）
└── docs\                     ← ドキュメント（参照用）
```

`PC_Optimizer.ps1` と `Run_PC_Optimizer.bat` と `modules/` フォルダは必ず**同じ親フォルダ**に置いてください。

## ネットワーク要件

本ツールは対象 PC からインターネットに直接接続する必要があります。

**常時接続が必要な機能:**
- Windows Update の確認・適用
- AI 診断（config.json の aiDiagnostics.enabled: true 時）

**オプション機能（無効時は通信なし）:**
- Slack/Teams 通知
- ServiceNow/Jira 起票

プロキシ環境の場合は PowerShell の `[System.Net.WebRequest]::DefaultWebProxy` を設定してください。

## config.json の初期設定

`config\config.json` を編集して動作をカスタマイズできます。

### AI 診断の有効化

Anthropic API キーを取得して以下のように設定します:

```json
{
  "aiDiagnostics": {
    "enabled": true,
    "apiKey": "sk-ant-...",
    "model": "claude-3-5-sonnet-20241022"
  }
}
```

### オプション機能の有効化

詳細は [config仕様.md](config仕様.md) を参照してください。

## 文字コード

スクリプトの保存形式は **UTF-8 with BOM** を推奨します。
UTF-8 without BOM の場合、PowerShell 5.1 で日本語が文字化けする場合があります。

VS Code で変換する場合：右下の `UTF-8` をクリック → `Save with Encoding` → `UTF-8 with BOM`

## PowerShell の実行ポリシー

`Run_PC_Optimizer.bat` は `-ExecutionPolicy Bypass` フラグで起動するため、
実行ポリシーの事前変更は不要です。

ポリシーを変更せずに直接 PS1 を実行したい場合：

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```
