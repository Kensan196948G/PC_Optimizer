# GUI イベント仕様

## 起動イベント

### EV-01 初期化

- PowerShell 実行エンジンを検出する
- `pwsh.exe` を優先し、未導入時のみ `powershell.exe` にフォールバックする
- `PC_Optimizer.ps1` の存在を確認する
- タスク一覧をロードする
- コマンドプレビューを初期化する

### EV-02 権限昇格

- 管理者権限がない場合、GUI 自身を昇格再起動する
- 昇格に失敗した場合はメッセージを表示して終了する

## 入力変更イベント

### EV-10 モード変更

- コマンドプレビューを再生成する
- `diagnose` 選択時は「診断向け選択」ボタンで task 20 を選びやすくする

### EV-11 タスク選択変更

- 選択タスク文字列を再計算する
- 進捗バーの分母を更新する
- コマンドプレビューを再生成する

### EV-12 オプション変更

- `WhatIf`、`FailureMode`、出力オプションの変更時にコマンドプレビューを再生成する
- `UseAnthropicAI` の見た目が他項目と重ならないようにレイアウトを維持する

## 実行イベント

### EV-20 実行開始

- 入力値をバリデーションする
- `ExportDeletedPaths` 選択時は `WhatIf` 必須とする
- `-NonInteractive -NoRebootPrompt` を付与して別プロセス起動する
- UTF-8 出力と装飾抑止を設定した一時ラッパースクリプトを生成し、実行エンジンからそのスクリプトを起動する
- 実行中は設定入力をロックする

### EV-21 標準出力受信

- ANSI 制御シーケンスを除去したうえでログビューへ追記する
- 進捗開始行と `[WhatIf]` / `[Tasks]` / `[Diagnose]` 行から現在タスクと完了数を更新する

### EV-22 標準エラー受信

- ANSI 制御シーケンスを除去したうえでログビューへ `[stderr]` 付きで追記する

### EV-23 プロセス終了

- 終了コードを表示する
- 設定入力を再度有効化する
- 最新レポートパスを探索する
- 実行用一時ラッパースクリプトを削除する

### EV-24 停止

- 実行プロセスを Kill する
- 状態を `停止` に更新する

## フォルダ操作イベント

### EV-30 レポートを開く

- `reports/` を Explorer で開く

### EV-31 ログを開く

- `logs/` を Explorer で開く

### EV-32 GUI ドキュメントを開く

- `docs/GUI/` を Explorer で開く

### EV-33 最新レポートを開く

- 最新 HTML / JSON レポートが存在する場合、既定アプリで開く
