# セキュリティ考慮事項

## 必要な権限

本ツールは**管理者権限**が必要です。これは以下のシステム領域にアクセスするためです。

- `C:\Windows\Temp` などのシステムディレクトリ
- `C:\Windows\SoftwareDistribution` などの Windows Update 関連ディレクトリ
- ディスク最適化 (`Optimize-Volume`, `defrag`)
- Windows Update サービスの操作
- Windows イベントログのクリア（`wevtutil`）
- システムファイル修復（`sfc`, `DISM`）
- 電源プランの変更（`powercfg`）

### 権限昇格の仕組み

`Run_PC_Optimizer.bat` は以下の手順で管理者権限を取得します。

```batch
net session >nul 2>&1          ← 管理者権限の確認
if %ERRORLEVEL% NEQ 0 (
    powershell Start-Process '%~f0' -Verb RunAs   ← UAC で昇格再起動
)
```

UAC プロンプトを表示してユーザーの同意を得てから昇格します。
サイレント昇格（UAC バイパス）は行いません。

## 実行ポリシー

BAT ファイルは `-ExecutionPolicy Bypass` フラグを使用します。
これはスクリプト実行のみに適用され、システム全体の実行ポリシーは変更しません。

## 削除操作の安全性

- すべての削除は `-ErrorAction SilentlyContinue` で実行されます
- ロック中のファイルはスキップされます（強制終了やプロセスキルは行いません）
- ユーザーのドキュメント・写真・重要ファイルは削除対象外です（[削除対象パス.md](削除対象パス.md) 参照）
- 削除前のバックアップ作成機能はありません

## Windows イベントログのクリア

`wevtutil cl Application` および `wevtutil cl System` でログをクリアします。
Security ログは**対象外**です（特権要件と監査要件のため）。

## Windows Update

`UsoClient.exe StartInstall` は Windows Update サービスへのトリガー送信であり、
更新プログラムの選択・フィルタリングは Windows Update サービスが行います。
本ツールはインストールする更新プログラムを指定・制御しません。

## ネットワーク通信

本ツールは外部ネットワーク通信を行いません。
Windows Update の通信はすべて Windows Update サービス（`wuauserv`, `UsoSvc`）が管理します。

## ログのセキュリティ

生成されるログファイルには以下の情報が含まれます。

| 含まれる情報 | 内容 |
|---|---|
| ホスト名 | `%COMPUTERNAME%` の値 |
| ユーザー名 | `%USERNAME%` の値 |
| OS バージョン | バージョン番号とマーケティング名 |
| CPU 情報 | モデル名・コア数 |
| メモリ容量 | 総物理メモリ (GB) |
| ディスク情報 | C ドライブの空き容量・総容量 |
| SSD 情報 | モデル名・シリアル番号・稼働状態 |

**SSD のシリアル番号**が含まれるため、ログファイルの共有・公開には注意してください。

ログファイルは `logs\` サブフォルダに平文テキストとして保存されます。
機密性が必要な場合は、実行後に安全な場所に移動するか削除してください。

## コード改変時の注意

スクリプトを改変する場合は以下に注意してください。

- `Remove-Item -Recurse -Force` に `-ErrorAction Stop` を指定すると、
  1 つのファイル削除失敗でタスク全体が中断します（デフォルト動作を変えないこと）
- `Restart-Computer -Force` は確認なしに即時再起動します。`Test-PendingReboot` の
  戻り値チェックとユーザー確認プロンプトを必ず維持してください
- `UsoClient.exe StartInstall` は実際の更新インストールを開始します。
  テスト環境では Windows Update タスクの呼び出しを無効化することを推奨します
