# リポジトリ運用方針

最終更新日: 2026-03-04

## 追跡対象（Git 管理に含める）
- `PC_Optimizer.ps1`
- `Run_PC_Optimizer.bat`
- `tests/` 配下のテストコード
- `docs/` 配下のドキュメント一式
- リリースノート（`docs/releases/`）

## 非追跡対象（Git 管理から除外）
- 実行時に生成されるログ/レポート（`logs/` 配下）
- ルート直下に一時的に生成されるログ（`PC_Optimizer_Log_*.txt`, `PC_Optimizer_Error_*.txt`）

## 補足
- 検証ログを証跡として共有する必要がある場合は、`docs/releases/` 配下へ要約を記載する。
- 生ログは原則コミットしない。

## リリースブランチ運用
- `release/v3.3.0` は保守専用ブランチとして扱う。
- 新機能開発は `master` から `feature/*` を作成して実施する。
- `release/v3.3.0` には原則として重大不具合修正のみを反映する。
- `master` の運用改善（CI/テンプレート/ドキュメント改善）を取り込む場合は、`master` から `release/v3.3.0` へ fast-forward で同期する。
