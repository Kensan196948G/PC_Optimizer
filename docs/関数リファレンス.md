# 関数リファレンス

## Write-Log

```powershell
Write-Log($msg)
```

メインログファイル（`logs/PC_Optimizer_Log_YYYYMMDDHHMM.txt`）に 1 行追記します。

| パラメータ | 型 | 説明 |
|---|---|---|
| `$msg` | string | ログに書き込む文字列 |

タイムスタンプは呼び出し側で付与します。

---

## Write-ErrorLog

```powershell
Write-ErrorLog($msg)
```

エラーログファイル（`logs/PC_Optimizer_Error_YYYYMMDDHHMM.txt`）に追記します。
先頭行に `[ERROR] yyyy/MM/dd HH:mm:ss` のタイムスタンプを自動挿入します。

| パラメータ | 型 | 説明 |
|---|---|---|
| `$msg` | string | エラー詳細文字列（`catch` ブロックの `$_` を推奨） |

---

## Show

```powershell
Show($msg, $color = "White")
```

色付きでコンソールに出力します。ログファイルには書き込みません。

| パラメータ | 型 | デフォルト | 説明 |
|---|---|---|---|
| `$msg` | string | — | 表示する文字列 |
| `$color` | string | `"White"` | `Write-Host` の `-ForegroundColor` に渡す色名 |

使用例：

```powershell
Show "処理完了" Green
Show "警告: SSD 未検出" DarkGray
Show "エラーが発生しました" Red
```

## Progress-Bar

```powershell
Progress-Bar($msg, $percent)
```

コンソールにプログレスバーと説明を表示します。
PowerShell バージョンに応じて表示スタイルが変わります。

| パラメータ | 型 | 説明 |
|---|---|---|
| `$msg` | string | バーの右に表示するメッセージ |
| `$percent` | int | 進捗率（0 〜 100）。10% 単位で描画される |

| PS バージョン | 表示例 |
|---|---|
| 7.x 以降 | `[██████████] 一時ファイルの削除 完了` |
| 5.1 以下 | `[##########] 進捗: 一時ファイルの削除 完了` |

各タスクの開始時（0%）と終了時（100%）の 2 回呼ばれます。

## Try-Step

```powershell
Try-Step($desc, [ScriptBlock]$action)
```

最適化タスクをラップし、開始/完了/失敗のログ記録とプログレスバー表示を自動で行います。

| パラメータ | 型 | 説明 |
|---|---|---|
| `$desc` | string | タスクの説明（ログとバーに表示） |
| `$action` | ScriptBlock | 実行するタスクのコードブロック |

**動作**:
1. 開始時刻をログに記録
2. プログレスバー（0%）を表示
3. `$action` を実行
4. 成功時: 完了ログ + バー（100%）
5. 失敗時: `Write-ErrorLog` にエラー記録 + バー（100%）
6. **例外が発生しても後続タスクを継続する**（throw しない）

## Test-PendingReboot

```powershell
[bool] Test-PendingReboot()
```

再起動待ち状態かどうかをレジストリで確認します。
戻り値: `$true`（再起動必要）/ `$false`（不要）

確認するレジストリキー:

| キー | 意味 |
|---|---|
| `HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update\RebootRequired` | Windows Update による再起動要求 |
| `HKLM:\SYSTEM\CurrentControlSet\Control\Session Manager\PendingFileRenameOperations` | ファイル名変更待ち（インストーラーなどが設定） |

## Run-WindowsUpdate

```powershell
Run-WindowsUpdate()
```

COM API（`Microsoft.Update.Session`）で利用可能な更新を検索・一覧表示し、
ユーザーの確認（Y/N）を得てから `UsoClient.exe` で適用します。
`UsoClient.exe` が存在しない環境（Windows 8.1 以前）では処理をスキップしログに記録します。

**動作フロー**:
1. `UsoClient.exe` の存在確認（なければスキップ）
2. COM API で利用可能な更新を検索（`IsInstalled=0 and IsHidden=0`）
3. 更新一覧を表示（最大 10 件、超過分は件数を表示）
4. 更新が 0 件の場合は自動スキップ
5. ユーザーに Y/N で確認
6. `Y` の場合: `UsoClient.exe StartScan → StartDownload → StartInstall`
7. `N` の場合: スキップをログに記録

| 手順 | コマンド |
|---|---|
| 1 | `UsoClient.exe StartScan` |
| 2 | `UsoClient.exe StartDownload` |
| 3 | `UsoClient.exe StartInstall` |

> `UsoClient.exe` は Windows 10 以降にのみ存在します。
> インストール完了まで待機はしません。実際の更新適用には数分〜数十分かかる場合があります。
> COM API の検索は数十秒かかる場合があります。

## modules/Notification.psm1

### Send-SlackNotification

```powershell
Send-SlackNotification([hashtable]$diagResult, [string]$webhookUrl, [hashtable]$options = @{})
```

Slack Incoming Webhook へスコア・評価・推奨アクションを POST します。

| パラメータ | 型 | 説明 |
|---|---|---|
| `$diagResult` | hashtable | 診断結果（スコア・評価・推奨アクションを含む） |
| `$webhookUrl` | string | Slack Incoming Webhook URL |
| `$options` | hashtable | オプション設定（省略可） |

スコアに応じて attachment の色を自動設定します。

| スコア | 色 |
|---|---|
| 80 以上 | `good`（緑） |
| 50 以上 | `warning`（黄） |
| 50 未満 | `danger`（赤） |

PS7 では `Invoke-RestMethod`、PS5.1 では `Invoke-WebRequest` + UTF-8 デコードで送信します。

戻り値: `@{ status: "Sent"|"Skipped"|"Failed"; provider: "slack" }`

### Send-ServiceNowIncident

```powershell
Send-ServiceNowIncident([hashtable]$diagResult, [hashtable]$config)
```

PC スコアをもとに ServiceNow インシデントを自動起票します。
`config.scoreThreshold` 未満のスコアの場合のみ起票します。

| パラメータ | 型 | 説明 |
|---|---|---|
| `$diagResult` | hashtable | 診断結果（スコアを含む） |
| `$config` | hashtable | ServiceNow 接続設定（instanceUrl, apiToken, scoreThreshold 等） |

スコアに応じた urgency を自動設定します。

| スコア | urgency | 重大度 |
|---|---|---|
| 0〜24 | 1 | Critical |
| 25〜49 | 2 | High |
| 50〜74 | 3 | Moderate |
| 75〜 | 4 | Low |

戻り値: `@{ status: "Created"|"Skipped"|"Failed"; provider: "servicenow" }`

## modules/Orchestration.psm1

### Convert-HookEntryToSiemLine

```powershell
Convert-HookEntryToSiemLine([hashtable]$entry, [string]$format = 'jsonl')
```

Hook エントリを SIEM 形式に変換します。`Export-ModuleMember` で公開済みのためテストから直接呼び出せます。

| パラメータ | 型 | デフォルト | 説明 |
|---|---|---|---|
| `$entry` | hashtable | — | Hook エントリ（status, name, timestamp 等を含む） |
| `$format` | string | `'jsonl'` | 出力形式。`jsonl` / `cef` / `leef` を指定可能 |

severity はエントリの成否で自動設定されます。

| 状態 | severity |
|---|---|
| 失敗 | 7 |
| 成功 | 3 |

戻り値: SIEM フォーマット文字列

### Export-HookSiemLines

```powershell
Export-HookSiemLines([array]$entries, [hashtable]$siemConfig)
```

Hook エントリを `logs/siem/` フォルダの SIEM ファイルに追記します。
`siemConfig.enabled = false` の場合は処理をスキップします。

| パラメータ | 型 | 説明 |
|---|---|---|
| `$entries` | array | Hook エントリの配列 |
| `$siemConfig` | hashtable | SIEM 設定（enabled, format 等） |

出力ファイル名は形式に応じて決まります（例: `siem_YYYYMMDD.jsonl`、`.cef`、`.leef`）。

戻り値: なし
