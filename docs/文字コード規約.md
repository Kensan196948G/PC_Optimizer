# 文字コード統一規約 v1.0

## 目的

Windows 11 / PowerShell 5.1 / PowerShell 7 / GitHub / CI 環境間での
日本語文字化け事故を防止する。

---

## 基本原則

| 種類 | 文字コード | 備考 |
|---|---|---|
| PowerShell 7 スクリプト | UTF-8 (BOM なし) | 標準 |
| PowerShell 5.1 スクリプト | UTF-8 BOM 付き | 明示指定必須 |
| ログファイル（PS7 出力） | UTF-8 (BOM なし) | `utf8NoBOM` を指定 |
| ログファイル（PS5.1 出力） | UTF-8 BOM 付き | `UTF8` を指定（BOM 自動付与） |
| JSON | UTF-8 (BOM なし) | `charset=utf-8` 明示 |
| HTML | UTF-8 | `<meta charset="utf-8">` 必須 |
| CSV | UTF-8 BOM 付き | Excel 互換のため |
| .bat | Shift-JIS (CP932) + CRLF | Windows コマンドプロンプト安定動作優先 |
| Markdown / テキスト | UTF-8 (BOM なし) | |

---

## 禁止事項

- `Out-File` / `Set-Content` / `Add-Content` の `-Encoding` 未指定
- UTF-16 ログファイルの生成
- 混在追記（UTF-16 ファイルへの UTF-8 追記）
- Git へ UTF-16 ファイルを Push
- `Out-File` のデフォルト（PS5.1 では UTF-16）への依存
- BAT ファイルへの LF のみの行末コード（cmd.exe でブロック解析が失敗する）

---

## 実装ルール

### PowerShell でのファイル出力

```powershell
# NG — エンコーディング未指定（PS5.1 では UTF-16 になる）
Out-File -FilePath $path -InputObject $content

# OK — 明示指定
Out-File    -FilePath $path -InputObject $content -Encoding UTF8
Set-Content -Path $path -Value $content -Encoding UTF8
Add-Content -Path $path -Value $content -Encoding UTF8
```

### PS バージョン別エンコーディング変数の使い方

```powershell
# バージョン検出は最初に実施
$psver       = $PSVersionTable.PSVersion.Major
$isPS7Plus   = ($psver -ge 7)

# エンコーディングを 1 箇所で決定し変数化する
# PS5.1: 'UTF8'      → UTF-8 with BOM   (メモ帳・Excel で文字化けしない)
# PS7.x: 'utf8NoBOM' → UTF-8 without BOM (IETF 標準)
$logEncoding = if ($isPS7Plus) { 'utf8NoBOM' } else { 'UTF8' }

# 以降は $logEncoding を使う
Add-Content -Path $logPath -Value $msg -Encoding $logEncoding
```

### BAT ファイルの保存（Shift-JIS + CRLF）

```powershell
# Shift-JIS + CRLF で BAT ファイルを書き込む
$sjis    = [System.Text.Encoding]::GetEncoding(932)
$content = $lines -join "`r`n"   # 明示的 CRLF（cmd.exe 必須要件）
[System.IO.File]::WriteAllBytes($batPath, $sjis.GetBytes($content))
```

### JSON 出力

```powershell
$obj | ConvertTo-Json | Out-File -FilePath $jsonPath -Encoding UTF8
```

### CSV 出力（Excel 互換）

```powershell
$data | Export-Csv -Path $csvPath -Encoding UTF8 -NoTypeInformation
# PS5.1 は -Encoding UTF8 で BOM 付き → Excel で直接開ける
```

## ソースファイルの保存形式

| エディタ | 操作 |
|---|---|
| VS Code | 右下エンコーディング → `Save with Encoding` → `UTF-8 with BOM`（PS5.1 向け） |
| メモ帳 | 「名前を付けて保存」→ エンコード「UTF-8 BOM 付き」 |
| PowerShell ISE | デフォルトで UTF-8 with BOM 保存 |

## .gitattributes 推奨設定

```gitattributes
# PowerShell スクリプト: LF 統一（クロスプラットフォーム対応）
*.ps1   text eol=lf
*.psm1  text eol=lf
*.psd1  text eol=lf

# BAT: Windows CRLF を維持
*.bat   text eol=crlf

# JSON / Markdown: LF
*.json  text eol=lf
*.md    text eol=lf
```

## CI 検査項目

- UTF-16 ファイルの検出（BOM バイト確認）
- BOM 混在ファイルの検出
- BAT ファイルの CRLF 統一確認（LF のみは cmd.exe で失敗）
- `-Encoding` 未指定の `Out-File` / `Set-Content` / `Add-Content` の Grep 検出

```powershell
# CI 用: Encoding 未指定のコードを検出
Select-String -Path "*.ps1" -Pattern "(Out-File|Set-Content|Add-Content)(?!.*-Encoding)" -Recurse
```

---

## 結論

**暗黙のデフォルトは禁止。すべて明示する。**

エンコーディングを明示しないコードはレビューで差し戻す。
